마크업 가이드

이미지 가이드

기본 규칙

  • 색상 정보를 가진 영역만 정확히 잘라내는 것을 기본으로 하되, 개발이나 코딩상의 이슈로 규격을 가지는 경우 여백을 두고 슬라이싱하는 것을 허용한다.
  • 포토샵 Save for Web 메뉴에서 이미지에 최적화된 포맷으로 저장하여 성능 향상을 도모한다.

이미지 타입

GIF
  • 256색 이하의 색상으로 표현되는 아이콘, 이미지.
  • 애니메이션 이미지 또는 투명한 부분까지 저장이 가능하나 반투명한 알파값은 표현 불가.
  • 투명한 부분과 이미지의 경계가 뚜렷한 경우 사용 가능
JPG
  • 사진, 그라데이션 등 많은 색상 정보를 포함하는 이미지에 적합.
  • 육안으로 화질의 손실이 느껴지지 않는 선에서 최대한 압축률을 높여 저장하는 것을 권장 (60~80)
PNG
  • 알파값 표현이 가능하므로 투명/반투명한 이미지를 저장할 때 적합
  • 화질의 손실이 거의 없는 반면, 압축률이 낮은 편이므로 꼭 필요할 때 사용

이미지 용량 및 크기

PC Web
  • 최대 사이즈 : 가로 1024px X 세로 1024px
    (모바일환경에서 PC 웹에 접속하였을 때, 다운로드된 이미지의 사이즈가 1024px이 넘어갈 때 해당 이미지가 올바르게 표시되지 않으므로 PC 웹에서 가로/세로 사이즈의 제한을 둔다.)
Mobile Web
  • CSS image : 스프라이트 된 이미지의 용량이 60KB 를 넘지 않도록 한다.
  • promotion image : 스프라이트 된 이미지의 용량이 400KB 를 넘지 않도록 한다.

이미지 네이밍 가이드

  • 영문,숫자,언더바(_)로 작성한다.
  • 숫자, 영문 대문자 시작할 수 없다.
  • 두단어의 조합일경우 언더바(_)로 구분한다.
  • 예약어를 근거로 작성한다.
  • 임시이미지에 한해 @를 허용한다.
  • 예약어를 근거로 작성한다.

이미지 공통 예약어

  • bg_*.확장자 : 배경
  • btn_*.확장자 : 버튼
  • ico_*.확장자 : 개별 작은 아이콘
  • icon_*.확장자 : 아이콘 묶음
  • h2_*.확장자 : 2depth 상단 h2태그
  • menu_*.확장자 : gnb menu
  • tit_*.확장자 : 타이틀 형태
  • img_*.확장자 : 이미지 형태
  • text_*.확장자 : 텍스트 형태
  • @banner_*.확장자 : 임시 샘플 배너 이미지
  • @smp_*.확장자 : 임시 샘플 이미지
  • *_on.확장자 : 활성화
  • *_off.확장자 : 비활성화

이미지 태그 사용 유의사항

  • 이미지 src 로 코딩할때는 절대경로로 잡아준다.
  • 이미지 src 뒤에는 alt="" 를 꼭 넣어주어 대체텍스트역할을 해준다.
  • 이미지 태그 끝에는 />(종로태그) 로 닫아준다.
  • 이미지 네이밍시에는 />예약어를 고려한다.
