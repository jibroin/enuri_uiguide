의미론적 마크업(Semantic Markup)

컴퓨터가 이해할 수 있는 정의된 의미를 기반으로 의미적 상호운용성을 실현하여, 인간과 컴퓨터 모두 잘 이해할 수 있는 웹을 만드는 것을 목표로 한다.

  • 시멘틱 HTML은 정확하고 접근이 용이한 컨텐츠의 데이타 구조 형성.
  • 무의미하게 사용하던 table 레이아웃을 배제하고, 의미를 살린 구조의 태그를 사용.
    <h1~h4>, <header>, <section>, <blockqoute>, <strong> 등

    ※ h5, h6는 파편화의 여지가 있으므로, 가급적 사용하지 않는다.

  • 가능한 가벼워야 하며, HTML, CSS, JS 는 각각 분리하여 마크업 구조(HTML)만으로 사용자에게 내용 파악이 가능하게 함.
  • CSS는 스타일 정보(시각적인 효과)를 제공.
  • HTML 표준을 준수하고 시맨틱한 문서를 작성하기 위해 노력하기는 하지만, 추가적인 노력이 필요하지 않은 범위 내에서만 한다.
    최대한 간결한 코드를 사용하도록 한다.

DOCTYPE 정의

HTML 문법은 해당 DTD의 명세에 기반하되, 확장성 및 호환성을 고려하여 XHTML1.0 Transitional 모드를 권장하여 작성한다.
(가급적 Strict 모드의 문법을 사용하는 것을 권장함)

웹일 경우 XHTML1.0(Transitional) 적용

<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd">

모바일 웹일 경우 HTML5 적용

<!DOCTYPE html>

예외 : 과거 작성된 XHTML4.1 은 이후 교체 예정

※ DOCTYPE 은 문서의 최상단에 위치하며, 그 위에 어떠한 코드와 공백을 허용하지 않는다.

주석 작성 규칙

주석은 장황하지 않게, 명료하고 정확한 명사형으로 표기한다.
<!-- LP 상품 목록 -->

주석 앞 뒤 코드 사이 공백을 넣어 표기한다. 예) <!--(공백)주석내용(공백)-->
<!-- LP 상품 목록 -->

주석의 시작과 끝에 동일한 문구로 기입하며, 닫는 주석은 달리 표시한다.
시작 <!-- LP 상품 목록 -->
끝 <!-- // LP 상품 목록 -->

들여쓰기

코드의 가독성을 높이기 위하여 들여쓰기를 사용하며, Tab 1개의 크기는 공백 4칸으로 설정한다.
마크업의 깊이에 따라 들여쓰지만 다음의 경우 들여쓰지 않는다.

- html, body 등 기본 태그와 body 태그 직속인 자식 태그

- thead, tbody, tfoot 의 자식 엘리먼트 tr

Attribute 작성

태그의 속성(Attribute)은 항상 큰 따옴표(“”)로 감쌉니다.

순서 속성 비고
1 id
2 class
3 name
4 data-*
5 src, for, type, href, value
6 title, alt
7 aria-*, role

마크업 작성 유의사항

  • 종료 태그 표기
  • 요소와 속성은 소문자로 표기
  • 속성값은 따옴표로 감싸서 표기
  • 특수문자는 Entity 코드로 변환
  • 모든 이미지와 이미지 맵에는 alt, title 속성을 부여해 주어 해당 이미지와 컨텐츠의 의미를 반영
  • 0을 제외한 모든 값들은 단위를 표기
